# 客户端与后端职责边界

## 判断顺序

遇到问题先按以下顺序定位：

1. 客户端是否请求了正确域名、路径、参数和 headers。
2. 后端是否返回了正确状态码、业务码和数据结构。
3. 本地缓存、SQLite/SQLCipher、登录态、Token、Socket 状态是否正常。
4. 是否涉及第三方能力，例如推送、支付、短信、地图、音视频 TURN。
5. 是否是部署、Nginx、数据库、Redis、服务进程或资源路径问题。

## 客户端负责

- Flutter 页面、路由、状态管理、组件渲染。
- Android/iOS 壳工程配置。
- APK 构建、安装、ADB 日志分析。
- 客户端 API baseUrl、WebSocket/TCP 客户端配置。
- 本地数据库、消息列表、会话列表、图片/视频预览、音视频客户端状态。
- 客户端第三方 SDK 是否启用、是否移除、是否占用权限。

## 后端负责

- API 服务接口实现、业务逻辑、数据库读写。
- 后台管理接口、权限、菜单、初始化数据。
- IM 服务、消息同步、群聊广播、离线消息。
- 文件上传、资源访问、缩略图生成。
- Nginx、SSL、systemd、Redis、MySQL、服务器防火墙。
- TURN/coturn 服务和音视频网络路径。

## 必须双方对齐的内容

- 登录、注册、验证码、Token 失效规则。
- 消息同步接口、游标、分页、ACK、重试。
- 上传接口字段、返回 URL、资源域名、缩略图策略。
- 音视频信令、房间状态、超时、挂断、忙线、异常恢复。
- 管理后台配置项哪些由服务端下发，哪些由客户端内置。
