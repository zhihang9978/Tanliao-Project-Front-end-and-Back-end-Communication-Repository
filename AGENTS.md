# AI 协作规则

## 角色边界

### Codex

Codex 负责客户端/前端：

- Flutter 客户端源码、Android APK 构建、客户端 UI、客户端配置。
- 客户端日志、ADB 日志、Flutter 性能、WebRTC 客户端状态机。
- 判断问题是否由客户端引起，并给出客户端修复方案。

### 后端 AI

后端 AI 负责服务器端：

- API 服务、后台服务、数据库、Redis、Nginx、systemd、部署脚本。
- IM 服务、消息同步、资源服务、TURN/coturn、服务器日志。
- 后端接口契约、数据初始化、权限和管理后台服务端逻辑。

## 协作要求

- Codex 不直接修改后端服务；如需后端变更，先在本仓库写清楚约定。
- 后端 AI 不直接修改客户端源码；如需客户端变更，先在本仓库写清楚约定。
- 涉及接口字段、路径、状态码、错误码、分页、上传、推送、音视频信令的改动，必须记录到 `docs/api-contract-log.md`。
- 涉及部署状态、账号初始化、联调结论的改动，必须更新 `PROJECT_STATE.md`。
- 所有问题先定位边界：客户端问题、后端问题、配置问题、第三方依赖问题、联调契约问题。

## 安全规则

禁止提交：

- SSH 密码、服务器密码、数据库密码。
- API Secret、AppSecret、Token、证书私钥。
- 用户隐私数据、真实聊天记录、真实手机号。

可以提交：

- 域名、接口路径、非敏感端口说明。
- 脱敏日志、复现步骤、错误码、截图说明。
- 非敏感测试账号规则，例如开发环境固定验证码说明。
